## 飞书常见问题处理

### HTTP 400 输入长度超限

**错误信息：**
```
HTTP 400: <400> InternalError.Algo.InvalidParameter: Range of input length should be [1, 202752]
```

**原因：** 飞书API对输入长度有限制（最大202752字符），对话历史/上下文累积过长会触发此错误。

**处理方法：**
1. 发现这个错误 → 立即重启服务清理缓存
2. 连续两次错误后 → 无法聊天，必须重启

**清理命令：**
```bash
openclaw restart
```

**预防措施：**
- 控制对话历史长度
- 定期重启服务
- 避免在单次对话中处理过长的内容
