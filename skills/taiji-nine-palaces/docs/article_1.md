## 核心机制

### 1. 自动组队

米珞根据任务类型自动匹配宫位组合：

```python
# 一句话创建任务
milo.do("下载抖音视频并生成摘要")

# 自动组队：1宫采集 → 7宫验收 → 5宫交付
# 无需人工分配，自动流转
```

**任务类型映射表**：

| 任务 | 组队 | 流程 |
|------|------|------|
| 视频处理 | 1→7→5 | 采集→验收→交付 |
| 数据分析 | 1→3→7→5 | 采集→分析→验收→交付 |
| 内容创作 | 4→8→7→5 | 策略→创作→验收→交付 |
| 技术开发 | 3→7→5 | 开发→验收→交付 |

### 2. TDD闭环

每个任务都遵循 Red-Green TDD 方法论：

```
🔴 红灯 → 定义验收标准
    ↓
🚀 执行 → 宫位协作完成
    ↓
🟢 绿灯 → 检查验收通过
    ↓
📦 交付 → 返回结果
```

这确保了每个任务都有明确的标准和质量保障。

### 3. 阴阳平衡

系统持续监控各宫负载，保持动态平衡：

- 高负载宫位：暂时不分配新任务
- 低负载宫位：优先分配
- 失衡告警：自动通知调整
