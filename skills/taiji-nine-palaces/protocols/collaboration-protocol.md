## 一、协议概述

定义各宫位之间的协作接口，确保任务流转顺畅。

---

## 二、宫位协作接口

### 2.1 数据流向

```
        正转（创造价值）
             ↓
    ┌────────────────────┐
    │   1宫 数据采集     │ → 原始数据
    └────────┬───────────┘
             ↓
    ┌────────────────────┐
    │   3宫 技术处理     │ → 处理结果
    └────────┬───────────┘
             ↓
    ┌────────────────────┐
    │   7宫 质量验收     │ → 验收结果
    └────────┬───────────┘
             ↓
    ┌────────────────────┐
    │   5宫 交付/汇报    │ → 成果交付
    └────────────────────┘
```

### 2.2 宫位间通信协议

| 从宫 | 到宫 | 任务类型 | 消息格式 |
|------|------|----------|----------|
| 1宫 | 3宫 | 原始数据 | `{type: 'raw_data', data: [...]}` |
| 3宫 | 7宫 | 待验收 | `{type: 'for_validation', deliverables: [...]}` |
| 7宫 | 5宫 | 验收结果 | `{type: 'validation_result', status: 'PASS/FAIL'}` |
| 5宫 | 余总 | 成果交付 | `{type: 'delivery', summary: '...'}` |
| 4宫 | 8宫 | 营销素材 | `{type: 'marketing_asset', content: '...'}` |
| 8宫 | 7宫 | 内容验收 | `{type: 'content_review', content: '...'}` |
| 6宫 | 5宫 | 监控报告 | `{type: 'monitoring_report', alerts: [...]}` |
| 9宫 | 4宫 | 行业数据 | `{type: 'industry_data', insights: [...]}` |

## 三、验收协议

### 3.1 验收请求格式

```json
{
  "request_id": "uuid",
  "from_palace": 3,
  "task_type": "code",
  "deliverables": ["file1.py", "test_results.json"],
  "standards": {
    "must_have": ["代码文件", "测试通过"],
    "quality_checks": ["语法正确", "覆盖率>80%"]
  },
  "priority": "normal"
}
```

### 3.2 验收响应格式

**通过（绿灯）**:
```json
{
  "status": "PASS",
  "request_id": "uuid",
  "deliver_to": 5
}
```

**失败（红灯）**:
```json
{
  "status": "FAIL",
  "request_id": "uuid",
  "return_to": 3,
  "retry_suggested": true
}
```

### 3.3 任务类型验收标准

| 任务类型 | 执行宫 | 验收标准 |
|----------|--------|----------|
| download | 1宫 | 文件存在、大小>0 |
| video_summary | 1宫→3宫 | 有标题、摘要、关键点 |
| code | 3宫 | 代码文件存在、测试通过 |
| brand_analysis | 4宫 | 报告完整、数据准确 |
| monitor | 6宫 | 数据完整、告警正常 |
| article | 8宫 | 内容合规、格式正确 |
| research | 9宫 | 来源可靠、结论清晰 |

## 四、风险等级判定

| 风险等级 | 触发条件 | 处理方式 |
|----------|----------|----------|
| LOW | 内部任务 | 自动通过 |
| MEDIUM | 跨宫协调 | 通知相关宫 |
| HIGH | 外部发布 | L1人工审批 |
| CRITICAL | 数据删除/权限变更 | 余总审批 |

## 五、协作场景

```
1宫下载视频 → 7宫验收 → 3宫转录 → 7宫验收 → 5宫交付
```

```

```

### 场景3：品牌分析

```
1宫采集数据 → 4宫分析 → 7宫验收 → 5宫交付
```

### 场景4：营销内容

```
4宫策略 → 8宫创作 → 7宫内容审核 → 5宫发布
```
