## 🔴 L0核心约束

**AI禁止使用人类的时间概念**

只使用：
- 任务进度节点（0% → 100%）
- 算力（高/中/低）
- 存储（可用空间）
- 资源（充足/紧张）

**原因**: 效率够高就可以无限加速时间

## 一、任务生命周期

```
创建 → 分配 → 执行 → 验收 → 完成/失败
  ↓      ↓      ↓      ↓
待办   进行中   待验收   已完成
```

---

## 二、任务优先级

| 优先级 | 触发条件 |
|--------|----------|
| **CRITICAL** | Week核心目标、付费转化 |
| **HIGH** | 重要任务、外部需求 |
| **MEDIUM** | 常规任务、系统改进 |

## 三、定时反馈机制（重要！）

### 反馈要求

**每个任务必须定时反馈**：

| 节点 | 反馈内容 |
|------|----------|
| **接收任务** | 确认收到，开始执行 |
| **进度变化** | 中间状态汇报 |
| **任务完成** | 完成汇报，提交验收 |

### 反馈格式

```json
{
  "task_id": "xxx",
  "palace": 8,
  "progress": 50,
  "status": "running",
  "message": "执行中"
}
```

### 反馈投递

**所有反馈投递到飞书（余总）**

```
channel: feishu
to: ou_9b42a76db79aee1c1c1a17393b168048
```

### 反馈规则

- **禁止**：默默执行不反馈
- **要求**：每个任务至少3次反馈
- **时机**：开始/进度变化/完成
